## Таблица 1 — Соответствие требованиям методички

| Требование | Реализация в проекте | Файлы |

## Таблица 3 — REST API

| Метод | URL | Назначение |
|---|---|---|
| POST | `/api/auth/register` | Регистрация |

| GET | `/api/users/me` | Профиль пользователя |
| GET | `/api/recipes` | Список рецептов |

| PATCH | `/api/shopping-list/items/{itemId}` | Отметить продукт купленным |

| PATCH | `/api/admin/recipes/{id}/reject` | Отклонить рецепт |

## Таблица 4 — Основные сущности данных

| Сущность | Поля | Назначение |
|---|---|---|
| `UserProfile` | name, email, role, avatarUri, counters | Профиль и роль пользователя |

| `UserSettings` | syncEnabled, offlineCacheEnabled, notificationsEnabled, compactModeEnabled | Пользовательские настройки |
| `AdminStats` | usersCount, recipesCount, pendingRecipesCount, reportsCount | Данные для админ-панели |

## Таблица 5 — Распределение по PCMEF

| Слой | Android-клиент | Spring Boot сервер |
|---|---|---|
| Presentation | Compose-экраны, карточки, нижняя навигация | REST не относится к UI |
| Control | `RecipeViewModel`, `RecipeUiState` | `RecipeController`, `AuthController`, `ShoppingListController` |
| Mediator | `RecipeInteractor` | `RecipeService`, `AuthService`, `ShoppingListService` |
| Entity | `Recipe`, `Ingredient`, `UserProfile` | JPA-сущности рецептов, пользователей и покупок |
| Foundation | `RecipeRepository`, Retrofit, Room, SharedPreferences | Spring Data JPA repositories, PostgreSQL |

## Таблица 6 — Тестовые сценарии

| Сценарий | Шаги | Ожидаемый результат |
|---|---|---|

| Полная очистка покупок | Нажать «Очистить весь список» | Список покупок становится пустым |
| Избранное | Нажать «Добавить в избранное» | Состояние рецепта меняется |
