# JaCoCo

![Testing Scheme](../images/19-testing-scheme.png)

Backend подключает JaCoCo в `pom.xml`.

Команда проверки:

```bash
mvn test jacoco:report
```

Фактический результат на 03.06.2026:

| Метрика | Покрытие |
|---|---:|
| Instruction coverage | 69.34% |
| Branch coverage | 45.24% |
| Line coverage | 64.95% |
| Complexity coverage | 52.35% |

В набор входят 19 unit/smoke-тестов для слоёв Control, Mediator и Security. Порог методички `>40%` выполнен по строкам, инструкциям, ветвлениям и сложности.

## Состав тестов

| Класс теста | Проверяемая область |
|---|---|
| `AuthServiceTest` | регистрация, вход, роль администратора, неверный пароль |
| `JwtTokenServiceTest` | генерация JWT, проверка подписи, отклонение изменённого токена |
| `RecipeServiceTest` | поиск, одобрение, отклонение, пустой запрос |
| `ShoppingListServiceTest` | добавление, переключение, очистка покупок |
| `ControllerSmokeTest` | делегирование REST-контроллеров сервисам |

## Анализ покрытия

Покрытие строк 64.95% превышает минимальное требование методички. Наиболее важные участки backend-логики покрыты: авторизация, JWT, модерация рецептов и список покупок. Ниже покрытие у инфраструктурных и DTO-классов, что допустимо для учебного проекта, потому что они в основном содержат простые поля и конфигурацию.

## Команда воспроизведения

```bash
cd server
mvn test jacoco:report
```

HTML-отчёт формируется в `server/target/site/jacoco/index.html`.
