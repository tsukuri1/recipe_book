# Руководство администратора

![Admin Moderation Sequence](../images/18-admin-moderation-sequence.png)

1. Войти под `admin@ladushki.app`.
2. Открыть профиль и перейти в админ-панель.
3. Просмотреть жалобы пользователей.
4. Взять жалобу в работу или закрыть ее.
5. Одобрить новый рецепт или удалить некорректный.

## Вход администратора

Для демонстрации используется аккаунт `admin@ladushki.app` с паролем `1234`. После входа приложение получает роль `ADMIN`, и пользователю становятся доступны административные функции. Если войти обычным пользователем, админ-панель не должна открывать защищённые операции.

## Работа с рецептами на проверке

Рецепты, созданные пользователями, получают статус `PENDING`. Администратор должен проверить:

- заполнено ли название;
- корректны ли ингредиенты и единицы измерения;
- не нарушает ли описание правила каталога;
- подходит ли изображение рецепта;
- нет ли явного дубликата.

После проверки рецепт можно одобрить или отклонить. Одобренный рецепт получает статус `APPROVED`.

## Работа с жалобами

Жалоба может указывать на ошибку в ингредиентах, дубликат рецепта или некорректное описание. Администратор переводит жалобу в статус «В работе», проверяет карточку рецепта, затем закрывает жалобу или принимает дополнительное решение по рецепту.

## Рекомендации

Перед демонстрацией нужно убедиться, что backend запущен, JWT работает, а в приложении доступен тестовый админ-аккаунт. Для защиты удобно заранее подготовить один рецепт со статусом `PENDING`.
