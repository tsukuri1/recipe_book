# Обновлённые тесты

![Testing Scheme](../images/19-testing-scheme.png)

После рефакторинга добавлены unit/smoke-тесты для backend-слоёв:

| Тест | Что проверяет |
|---|---|
| `AuthServiceTest` | регистрация, вход, роли, ошибки авторизации |
| `JwtTokenServiceTest` | создание и проверка JWT |
| `RecipeServiceTest` | поиск, одобрение и отклонение рецептов |
| `ShoppingListServiceTest` | список покупок |
| `ControllerSmokeTest` | делегирование REST-контроллеров |

## Что изменилось после добавления тестов

До расширения тестового набора backend мог успешно компилироваться, но это не доказывало корректность бизнес-логики. После добавления тестов проверяются реальные сценарии: регистрация администратора, проверка пароля, подпись JWT, модерация рецепта, очистка списка покупок и делегирование контроллеров.

## Почему тесты находятся на backend

Методичка для мобильной траектории отдельно требует модульное тестирование сервера с покрытием выше 40%. Поэтому основной автоматический набор находится в `server/src/test`. Android-сценарии описаны в тест-плане и проверяются вручную через эмулятор или устройство.

## Результат

Команда `mvn test jacoco:report` запускает 19 тестов без ошибок. Покрытие строк составляет 64.95%, что выше требуемого порога.
