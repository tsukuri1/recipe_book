## Цель рефакторинга

Рефакторинг выполнялся не ради изменения внешнего поведения, а ради улучшения структуры проекта. Основная цель — сохранить разделение PCMEF, убрать смешивание UI и инфраструктуры, а также сделать backend-логику тестируемой без запуска всей системы.

![DTO Mapping Diagram](../images/17-dto-mapping-diagram.png)

## Основные направления

- отделение DTO и cache-моделей от доменных моделей;
- добавление JWT-сервиса вместо простого token payload;
- уточнение роли `TokenStore` на Android;
- расширение backend-тестов;
- документирование проблем и способов их устранения.
