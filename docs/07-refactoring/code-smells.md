# Исправленные проблемы

| Проблема | Исправление |
|---|---|
| Дубли ключей в LazyColumn | Добавлены уникальные префиксы ключей |
| Аватар терялся после перезапуска | Добавлен `ProfileStore` на SharedPreferences |
| Пустая кнопка «Одобрить» | Добавлена логика `approveRecipe` |
| Неполная очистка покупок | Добавлен endpoint `DELETE /shopping-list` |

## Подробное описание проблем

### Дублирование ответственности

На раннем этапе часть логики могла оказаться прямо в UI: например, формирование сообщений, работа с покупками или обработка профиля. Это ухудшает тестируемость, потому что Compose-экран становится слишком умным. Решение — переносить сценарии в ViewModel и Interactor.

### Нестабильная авторизация

Если токен не сохраняется после входа, последующие запросы к защищённым endpoint не смогут пройти. Для решения добавлен `TokenStore`, а backend выдаёт подписанный JWT.

### Неполная документация тестов

Изначально команда `mvn test` проверяла только компиляцию. После добавления unit-тестов и JaCoCo отчёт стал подтверждать выполнение требования методички о покрытии выше 40%.

## Вывод

Исправленные code smells связаны не только с отдельными багами, но и с архитектурой. После рефакторинга проект лучше соответствует PCMEF: UI стал тоньше, бизнес-логика проверяемее, а инфраструктура отделена от доменных сценариев.
