## Назначение раздела

Управление проектом показывает, как работа была разложена на этапы, какие задачи выполнялись и как оценивались трудозатраты. Для курсового проекта это важно, потому что методичка требует показать жизненный цикл разработки, а не только итоговый код.

![Testing Scheme](../images/19-testing-scheme.png)

## Связь с этапами методички

WBS показывает состав работ, диаграмма Ганта — порядок выполнения, COCOMO — примерную оценку трудозатрат. Вместе эти документы подтверждают, что проект проходил этапы анализа, проектирования, реализации, тестирования и оформления.

## Почему это важно

Для программной инженерии важно показать управляемость разработки. Даже небольшой учебный проект содержит риски: настройка Android SDK, запуск PostgreSQL, синхронизация клиента и backend, подготовка диаграмм и тестов. Планирование помогает показать, что эти задачи были учтены заранее.

## Вывод

Раздел управления проектом связывает техническую часть с организационной: он показывает объём работ, последовательность этапов и примерную стоимость разработки в трудозатратах.
