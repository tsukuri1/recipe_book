# Иерархическая структура работ

![Component Diagram](../images/07-component-diagram.png)

| Этап | Работы |
|---|---|
| Анализ | Тема, требования, сценарии |
| Проектирование | PCMEF, модели, API, UI |
| Реализация | Android, backend, local storage |
| Тестирование | Сборки, сценарии, исправления |
| Документация | README, таблицы, руководства |

## Подробная WBS

1. Анализ
   - выбор темы;
   - описание проблемы;
   - определение акторов;
   - формирование глоссария.

2. Требования
   - функциональные требования;
   - нефункциональные требования;
   - use case;
   - трассировка требований.

3. Проектирование
   - PCMEF;
   - ER-модель;
   - REST API;
   - UI-концепция.

4. Реализация
   - Android-экраны;
   - ViewModel и состояние;
   - backend-контроллеры;
   - сервисы и JPA;
   - JWT и security.

5. Проверка
   - unit-тесты backend;
   - JaCoCo;
   - ручные сценарии Android;
   - исправление ошибок.

6. Сдача
   - README;
   - docs;
   - пояснительная записка;
   - архив проекта.

## Вывод

WBS показывает, что проект выполняется как инженерная работа с этапами, а не как набор несвязанных файлов.
