# Матрица стейкхолдеров

| Стейкхолдер | Интерес | Влияние | Ожидания |
|---|---|---:|---|
| Пользователь | Быстро хранить и находить рецепты | Высокое | Удобный каталог, поиск, покупки |
| Администратор | Контроль качества рецептов | Среднее | Очередь модерации и жалобы |
| Преподаватель | Проверка соответствия методичке | Высокое | PCMEF, документация, тесты, API |
| Разработчик | Поддерживаемость проекта | Высокое | Чёткие слои, тестируемость, понятная структура |

## Анализ влияния

Главный стейкхолдер — конечный пользователь. Именно его сценарии определяют интерфейс: каталог должен быть быстрым, карточка рецепта понятной, а список покупок — удобным в реальном магазине. Администратор влияет на качество контента: без модерации общий каталог может заполниться неполными или ошибочными рецептами.

Преподаватель оценивает проект не только как приложение, но и как инженерный артефакт. Поэтому документация, PCMEF-слои, тесты, REST API и структура репозитория имеют такое же значение, как и работающий интерфейс.

## Коммуникационные потребности

- Пользователю нужна простая инструкция и понятные экранные сценарии.
- Администратору нужно руководство по модерации.
- Преподавателю нужна трассировка требований до реализации.
- Разработчику нужны архитектурные решения и описание слоёв.
