## Назначение раздела

Этап 00 нужен, чтобы показать, что проект начинается не с кода, а с понимания предметной области. Для приложения «Ладушки» предметная область связана с домашним приготовлением еды: пользователь хранит рецепты, выбирает блюда на ближайшие дни, переносит ингредиенты в список покупок и постепенно формирует личную кулинарную базу.

![Use Case Diagram](../images/01-use-case-diagram.png)

В разделе описаны заинтересованные стороны, бизнес-цели, ограничения и экономический смысл проекта. Эти материалы используются дальше при формировании требований, проектировании архитектуры PCMEF и выборе функциональности MVP.

## Вывод по этапу

Проект имеет понятную пользовательскую ценность: он уменьшает ручную работу с рецептами и покупками, объединяя каталог блюд, профиль, список продуктов и модерацию в одном мобильном приложении. Для курсового проекта тема достаточно сложная, потому что включает мобильный интерфейс, REST API, локальное кэширование, безопасность и административные сценарии.
