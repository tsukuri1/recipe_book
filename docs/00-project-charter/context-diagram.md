# IDEF0 контекст

![Component Diagram](../images/07-component-diagram.png)

## A-0. Управление кулинарными рецептами

| Блок | Описание |
|---|---|
| Входы | Рецепты, ингредиенты, пользовательские данные, фотографии блюд |
| Управление | Методичка курса, правила модерации, требования безопасности |
| Механизмы | Android-клиент, Spring Boot backend, PostgreSQL, Room |
| Выходы | Каталог рецептов, список покупок, профиль пользователя, отчёты администратора |

Главная функция системы — хранить рецепты, поддерживать поиск и модерацию, а также формировать список покупок по выбранным блюдам.

## Декомпозиция A0

| Подфункция | Назначение |
|---|---|
| A1. Управлять пользователями | Регистрация, вход, хранение роли, получение профиля |
| A2. Управлять рецептами | Каталог, карточка рецепта, создание, редактирование, удаление |
| A3. Формировать покупки | Перенос ингредиентов в чек-лист и отметка купленных продуктов |
| A4. Модерировать контент | Просмотр новых рецептов, одобрение, отклонение, жалобы |
| A5. Синхронизировать данные | REST API, локальный Room-кэш, сохранение настроек |

## Ограничения

Система должна работать как мобильное приложение с backend на Java. Для разработки используется локальный сервер Spring Boot, поэтому Android-эмулятор обращается к нему через `10.0.2.2`. Безопасность реализуется через JWT, а пароли пользователей хранятся в виде BCrypt-хеша.

## Вывод

IDEF0-контекст показывает границы проекта: пользователь взаимодействует с Android-клиентом, клиент обращается к REST API, backend работает с PostgreSQL, а часть данных сохраняется локально для offline-сценариев.
