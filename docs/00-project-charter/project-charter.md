# Паспорт проекта

![PCMEF Architecture](../images/06-pcmef-architecture.png)

| Поле | Значение |
|---|---|
| Название | Ладушки |
| Тема | Мобильное приложение для ведения кулинарных рецептов |
| Цель | Упростить хранение рецептов и составление списка покупок |
| Пользователи | Обычные пользователи и администратор |
| Платформа | Android + Spring Boot |
| Основные функции | Рецепты, покупки, профиль, админ-панель |

## Проблема

Домашние рецепты часто хранятся в разных местах: заметках, мессенджерах, фотографиях, бумажных блокнотах. Из-за этого пользователю сложно быстро найти нужное блюдо, вспомнить ингредиенты и составить список покупок. Дополнительная проблема — качество общих рецептов: если пользователи добавляют свои материалы, требуется модерация.

## Решение

«Ладушки» объединяет каталог рецептов, создание собственных блюд, список покупок, профиль и админ-панель в одном мобильном приложении. Backend отвечает за хранение данных, безопасность, REST API и модерацию, а Android-клиент отвечает за удобный пользовательский интерфейс и локальное кэширование.

## Цели проекта

1. Реализовать мобильное приложение с 5+ экранами и навигацией.
2. Реализовать Spring Boot backend с REST API и Swagger.
3. Обеспечить вход по JWT и разграничение ролей.
4. Добавить offline-кэширование через Room.
5. Подготовить документацию по этапам методички.

## Критерии успешности

Проект считается успешным, если пользователь может войти, открыть каталог, найти рецепт, создать рецепт, сформировать список покупок и изменить профиль, а администратор может модерировать рецепты. Для backend должны проходить unit-тесты с покрытием выше 40%.
