# SWOT-анализ

| Сторона | Описание |
|---|---|
| Strengths | Удобный UI, фото рецептов, КБЖУ, покупки |
| Weaknesses | Серверная синхронизация требует запущенного PostgreSQL и настройки Android base URL |
| Opportunities | Расширение до полноценного сервиса рецептов |
| Threats | Конкуренция с крупными кулинарными приложениями |

## Strengths

Сильная сторона проекта — сочетание нескольких практичных сценариев в одном приложении. Пользователь не просто читает рецепт, а может добавить ингредиенты в покупки, отметить избранное, сохранить профиль и работать с локальными данными. Для курсовой работы важно, что проект показывает и мобильный UI, и backend, и API, и безопасность.

## Weaknesses

Слабое место учебной версии — необходимость локальной настройки backend и PostgreSQL. Для демонстрации нужно заранее запустить сервер, создать базу `ladushki` и указать корректный base URL для Android-эмулятора. Кроме того, offline-режим ориентирован на кэширование уже загруженных данных, а не на полноценную двустороннюю синхронизацию конфликтов.

## Opportunities

Проект можно развить до семейного планировщика питания: недельное меню, совместный список покупок, подбор блюд по диете, интеграция с доставкой продуктов, рейтинги рецептов, комментарии и push-уведомления.

## Threats

Основная угроза — высокая конкуренция с готовыми сервисами рецептов. Чтобы приложение было полезным, нужно развивать уникальные сценарии: удобное создание личных рецептов, offline-доступ, быстрые покупки и качественная модерация.
