## Что реализовано

В проекте реализованы две части: Android-клиент и Spring Boot backend. Android-клиент содержит экраны, ViewModel, доменные модели, fake-репозиторий для демонстрации, сетевой репозиторий, Retrofit API и Room-кэш. Backend содержит контроллеры, сервисы, JPA-сущности, репозитории и security-слой.

## Принцип реализации

Код строится вокруг сценариев пользователя. Например, сценарий добавления рецепта начинается в `AddRecipeScreen`, проходит через `RecipeViewModel.saveRecipe`, затем через `RecipeInteractor` и `RecipeRepository`. На backend аналогично: HTTP-запрос принимает контроллер, сервис применяет бизнес-правила, репозиторий сохраняет данные.

## Что смотреть при проверке

При проверке реализации удобно двигаться от пользовательского сценария к коду. Например, сценарий «добавить ингредиенты в покупки» начинается на экране карточки рецепта, затем проходит через ViewModel, Interactor и Repository. Такой путь показывает, что слои не существуют только в документации, а реально используются в коде.

## Вывод

Реализация закрывает обязательное ядро мобильной траектории: Android-интерфейс, backend, REST API, локальное хранение, безопасность и тесты. Структура пакетов поддерживает дальнейшее расширение проекта без крупных переписываний.
