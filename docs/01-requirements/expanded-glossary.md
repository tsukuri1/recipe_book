# Расширенный глоссарий

| Термин | Определение |
|---|---|
| JWT | Подписанный токен доступа для защищённых REST-запросов |
| Offline cache | Локальное хранение данных в Room для работы без сети |
| Moderation | Проверка пользовательского рецепта администратором |
| Nutrition | КБЖУ: калории, белки, жиры, углеводы |
| PCMEF | Архитектурный шаблон Presentation-Control-Mediator-Entity-Foundation |
| Shopping list | Чек-лист продуктов, сформированный из ингредиентов рецепта |

## Термины интерфейса

| Термин | Определение |
|---|---|
| Главная | Первый экран после входа, где отображается приветствие и быстрые переходы |
| Каталог | Экран со списком рецептов, поиском и категориями |
| Карточка рецепта | Детальный экран с фото, ингредиентами, шагами и КБЖУ |
| Админ-панель | Экран администратора для статистики, жалоб и рецептов на проверке |
| Компактный режим | Пользовательская настройка, влияющая на плотность интерфейса |

## Термины backend

| Термин | Определение |
|---|---|
| Controller | Слой Control, принимающий HTTP-запросы |
| Service | Слой Mediator, где находятся бизнес-правила |
| Repository | Слой Foundation, отвечающий за доступ к данным |
| DTO | Объект передачи данных между API и клиентом |

Глоссарий нужен, чтобы одинаковые понятия использовались в требованиях, диаграммах, коде и пояснительной записке.
