# Use Case диаграмма

![Use Case Diagram](../images/01-use-case-diagram.png)

## Акторы

| Актор | Возможности |
|---|---|
| Пользователь | Просмотр рецептов, поиск, избранное, покупки, профиль, настройки |
| Администратор | Модерация рецептов, просмотр статистики, обработка жалоб |

## Перечень прецедентов

| Use Case | Актор | Цель |
|---|---|---|
| Войти в систему | Пользователь, администратор | Получить доступ к персональным функциям |
| Найти рецепт | Пользователь | Быстро подобрать блюдо |
| Открыть карточку рецепта | Пользователь | Посмотреть ингредиенты, шаги и КБЖУ |
| Добавить рецепт | Пользователь | Сохранить собственный рецепт |
| Добавить в покупки | Пользователь | Перенести ингредиенты в чек-лист |
| Изменить профиль | Пользователь | Обновить имя и аватар |
| Сохранить настройки | Пользователь | Изменить режимы приложения |
| Одобрить рецепт | Администратор | Опубликовать пользовательский рецепт |
| Рассмотреть жалобу | Администратор | Поддерживать качество каталога |

## Границы системы

Use Case диаграмма показывает только взаимодействие пользователя и администратора с приложением. Внутренние компоненты, такие как Room, PostgreSQL, Retrofit и Spring Data JPA, не являются акторами, потому что они входят в состав системы.
