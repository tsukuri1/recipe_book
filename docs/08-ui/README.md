## Цель UI-раздела

Раздел описывает не только внешний вид, но и пользовательскую логику экранов. Для мобильной траектории важно показать, что приложение имеет больше пяти экранов, навигацию, обработку состояний, формы ввода и связь с предметной областью.

![Navigation Diagram](../images/21-navigation-diagram.png)

## Основные принципы

- интерфейс должен быть понятен без технических терминов;
- главные действия доступны через нижнюю навигацию;
- карточки рецептов должны хорошо сканироваться;
- список покупок должен быть удобен в магазине;
- админ-панель должна отделяться от обычных пользовательских функций.

## Что демонстрировать

Для защиты лучше показать не отдельные экраны, а связанный путь пользователя: вход, поиск рецепта, открытие карточки, добавление ингредиентов в покупки, отметка купленного и изменение профиля. Затем можно войти администратором и показать модерацию.

## Вывод

UI поддерживает предметную область приложения: рецепты отображаются как карточки блюд, ингредиенты превращаются в список покупок, а настройки и профиль не перегружают основной сценарий приготовления.
